**【核心业务定义】**
1. **游戏映射**：砺刃使者(jordass)、勇者盟约(initiatived)、峡谷行动(esports) 均为 FPS 游戏。
2. **大盘定义**：“全平台/手游大盘”指所有游戏集合；“某游戏大盘”（如砺刃大盘）指该游戏下的所有活跃用户（非特定玩法）。
3. **默认活跃**：若问题未指定具体玩法，默认指“游戏登录活跃”。
4. **术语消歧**：“流水”可能指行为日志（dwd表），也可能指充值金额（Pay表），请根据上下文判断。

**【表与时间查询规范 (关键)】**
1. **表后缀规则**：
   - `_di` (增量表)：存储每日新增数据。查询一段时间需用 `dtstatdate BETWEEN 'start' AND 'end'`。
   - `_df` (全量/快照表)：存储截至当日的累积状态。查询一段时间的状态（如等级、留存），**通常只取时间段的最后一天** `dtstatdate = 'end_date'`。
2. **cbitmap 字段**：
   - 100位字符串，**最左侧(第1位)**代表当前统计日。
   - `1`=有行为，`0`=无行为。
   - 常用于计算留存/回流（如 `substr(cbitmap, 1, 7)` 判断近7天情况）。

**【SQL 优先原则】**
- **优先聚合**：如果存在 DWS 层（聚合表），优先使用 DWS 表，避免直接查询 DWD 层（明细表）。
